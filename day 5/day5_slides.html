Vocabulary Time! üìö

Git
A version control system that tracks changes to your files over time. Think of it like Google Docs' version history, but for code! It lets you save snapshots of your work and collaborate with others.

Real-world analogy: Git is like having a time machine for your code. You can go back to any point in time and see what your code looked like, or even restore it if you mess something up!

Vocabulary: HTML Tag üè∑Ô∏è

Tag (HTML)
A piece of code that tells the browser how to display content. Tags usually come in pairs with an opening tag like <p> and a closing tag like </p>. They're like instructions that wrap around your content.

Examples of HTML tags:

  • <h1> - Makes a big heading
  • <p> - Creates a paragraph
  • <strong> - Makes text bold
  • <img> - Adds an image (no closing tag!)

Fork and Clone: Make It Yours! üç¥

Fork
Creating your own copy of someone else's repository on GitHub. Like making a photocopy of a recipe so you can modify it.
Clone
Downloading a repository from GitHub to your computer. Like downloading a game so you can play it offline.

The Process:

  1. Fork on GitHub (makes your online copy)
  2. Clone to computer (downloads your copy)
  3. Make changes locally
  4. Push back to GitHub
  5. Create Pull Request (ask to merge changes)
