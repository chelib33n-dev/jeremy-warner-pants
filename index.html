üìñ Vocabulary Terms

‚ñº Day 1: Introduction & Binary Basics
Bit
The smallest unit of data in computing, representing a single 0 or 1.
Byte
A unit of digital information that consists of 8 bits.
Repository
A storage location for code and project files, often used for collaboration and version control (e.g., on GitHub).
‚ñº Day 2: Binary Math & Computer Basics
Operating System
The main software that manages a computer's hardware and software resources, and provides common services for computer programs. Examples include Windows, MacOS, Linux, Android, and iOS.
File Extension
The suffix at the end of a filename that indicates what type of file it is (for example, .docx for Word documents, .jpg for images, .exe for executable programs).
Hard Drive
A data storage device used for storing and retrieving digital information. It retains data even when the computer is turned off and is used to store files, programs, and the operating system.
RAM (Random Access Memory)
A type of computer memory that is used for temporary storage while a computer is running. It allows data to be read and written quickly, but all information is lost when the power is turned off.
‚ñº Day 3: Command Line & Web Basics
Gigabyte (GB)
A unit of digital storage equal to approximately 1 billion bytes (technically 1,024 megabytes). Think of it like a large box that can hold about 250 songs or a 30-minute HD video.
Command Line (Terminal)
A text-based way to control your computer by typing commands instead of clicking with a mouse. On Mac, it's called Terminal. On Windows, it's called Command Prompt or PowerShell.
HTML
HyperText Markup Language - The standard language for creating web pages. It uses tags to tell the browser what each part of the page is: headings, paragraphs, images, links, etc.
‚ñº Day 4: HTML & Git Hands-On
Open Source
Software with source code that anyone can inspect, modify, and share. It's like a recipe that everyone can see, improve, and use for free! Examples include Linux, Firefox, Android, and VS Code.
Git
A version control system that tracks changes in your code. Think of it like a super-powered "undo" button that remembers every change you've ever made to your files. Git is the tool, while GitHub is a website that uses Git.
